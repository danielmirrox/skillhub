## Часть 2. Детальная проработка разделов

### 1) Концепт

Проблема:
Студенческие хакатоны и проектная деятельность страдают от асимметрии информации.

1. Проблема "кота в мешке":
- В чатах поиска команды все пишут "знаю всё", но на деле уровень участников часто не соответствует заявленному.
- Это приводит к развалу команд посреди хакатона.

2. Проблема "спама":
- Сильные ребята (капитаны) получают десятки сообщений от новичков и не могут быстро отфильтровать релевантных.

3. Проблема новичка:
- Студенту сложно объективно оценить свой уровень (эффект Даннинга-Крюгера), из-за чего он подается не туда, где будет полезен.

Цифры (гипотетические для питча):
- 70% команд на студенческих хакатонах распадаются или не доходят до финиша из-за несовпадения ожиданий по скиллам.
- В среднем капитан тратит 4–6 часов на поиск и собеседование участников.

Ценностное предложение (Value Proposition):
- Для участника: объективная оценка навыков от AI + попадание в релевантные команды.
- Для капитана: экономия времени. Видно верифицированный рейтинг (например, 89/100), а не просто текст "хочу в команду".

ЦА:
- Студенты IT-специальностей
- Организаторы хакатонов
- Лидеры студенческих стартапов

### 2) Продуктовый подход

Бизнес-модель (Lean Canvas, Revenue Stream):

Freemium Model:
- Free:
  - Создание профиля
  - Генерация AI-рейтинга (1 раз в неделю)
  - Просмотр общей ленты (без контактов)
- Premium (Captain Mode): 299 руб/мес (или разовая покупка на хакатон)
  - Возможность писать первым пользователям с рейтингом 80+
  - Приоритетный показ профиля в ленте (подсветка)
  - Детальная расшифровка оценки от нейросети
  - Рекомендации по прокачке и зоны роста

Путь пользователя (CJM):
1. Вход: логин через GitHub/Telegram.
2. Онбординг: форма (стек, ссылки на проекты, опыт).
3. AI processing: данные отправляются в YandexGPT, пользователь ждет 5–10 секунд, получает бейдж типа "Frontend: 78/100".
  - PRO показывает ещё рекомендации по улучшению
4. Поиск:
- Сценарий A (ищу команду): листаю вакансии, вижу метч по баллам, откликаюсь.
- Сценарий B (ищу бойца): фильтр по рейтингу > 80, открытие контактов (если оплачено), сообщение кандидату.

### 4) Техническая архитектура

Схема компонентов:

[Client (React/Next.js)] <--> [API Gateway (Node.js/Express)] <--> [PostgreSQL (Users, Ratings)]
                                                    |
                                                    v
                                           [YandexGPT API (Yandex Cloud)]

Обоснование стека:
- Frontend: React + Vite + TailwindCSS
  - Скорость разработки, адаптивность под мобильные устройства
- Backend: Node.js
  - Единый язык с фронтендом, удобно шарить типы данных
- Database: PostgreSQL (Yandex Cloud Managed PostgreSQL)
  - Доступно из РФ, в одном месте с YandexGPT API, надёжное хранение данных
- Auth: Passport.js + GitHub OAuth
- AI: YandexGPT
  - Быстрый API, хорошее понимание русского языка

Реалистичный MVP-план на хакатон (24–48 часов):
1. Часы 0–6: база данных + авторизация (GitHub login)
2. Часы 6–12: интеграция YandexGPT, промпт-инжиниринг
3. Часы 12–18: верстка профиля с рейтингом и ленты пользователей
4. Часы 18–24: фильтрация по рейтингу, заглушка оплаты (эмуляция Premium)
5. Часы 24+: полировка, тестовые данные, подготовка демо
