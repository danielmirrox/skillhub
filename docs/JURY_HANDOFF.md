# SkillHub - Jury Handoff

Короткий operational-гайд для финальной сдачи, деплоя и live-demo.

## Что отправить жюри

Нужно сдать 3 ссылки:

- ссылка на презентацию
- ссылка на открытый GitHub-репозиторий
- домен или ссылку на работающий проект

## Что должно быть готово перед сдачей

- мобильная верстка проверена на смартфоне
- frontend открывается по публичному домену
- backend отвечает на `GET /health`
- GitHub OAuth callback указывает на боевой домен
- в `server/.env` заданы боевые переменные
- `COOKIE_SECURE=true`, если проект открыт по HTTPS
- PostgreSQL доступен и не падает в demo fallback
- презентация, домен и GitHub указывают на одну и ту же актуальную версию продукта

## Быстрый деплой

Рекомендуемый путь: Docker Compose на одной VM.

1. Скопировать репозиторий на сервер.
2. Подготовить `server/.env` из `server/.env.example`.
3. Проверить боевые значения:
   - `DATABASE_URL`
   - `CLIENT_URL`
   - `CLIENT_URLS`
   - `GITHUB_CLIENT_ID`
   - `GITHUB_CLIENT_SECRET`
   - `GITHUB_CALLBACK_URL`
   - `JWT_SECRET`
   - `COOKIE_SECURE`
   - `YANDEXGPT_*`
4. Запустить стек:

```bash
docker compose up -d --build
```

5. Проверить:
   - `http(s)://<domain>/`
   - `http(s)://<domain>/health` для backend proxy или `http(s)://<api-domain>/health`
   - GitHub login
   - `/match`
   - `/search`
   - `/teams`
   - `/applications`

Подробности по compose-разворачиванию: [DEPLOYMENT_COMPOSE.md](./DEPLOYMENT_COMPOSE.md)

## Что показать на демо

Короткий сценарий на защите:

1. Открыть главную.
2. Нажать GitHub login.
3. Показать экран `Подбор`.
4. Открыть профиль и рейтинг.
5. Открыть поиск участников.
6. Открыть команды.
7. Отправить заявку.
8. Показать входящие/исходящие заявки.

Если времени мало:

1. login
2. подбор
3. профиль
4. команды
5. заявки

## Что важно для мобильного показа

- проверить главную страницу
- проверить login page
- проверить `/match`
- проверить `/search`
- проверить `/teams`
- проверить `/profile`
- проверить, что меню и CTA доступны с мобильного

## Команды для локальной проверки

Frontend build:

```bash
cd client
npm run build
```

Smoke:

```bash
cd server
npm run smoke
```

## План B на защите

Если внешний GitHub OAuth временно недоступен:

- заранее держать открытым уже авторизованный браузерный сеанс
- иметь быстрый маршрут показа: `/match`, `/profile`, `/teams`, `/applications`

Если YandexGPT временно недоступен:

- честно сказать, что у проекта есть fallback scoring path
- показать уже рассчитанный рейтинг из seed/истории

## Перед выходом на сцену

- проверить домен со смартфона
- проверить, что login не ломает callback
- открыть презентацию заранее
- открыть GitHub-репозиторий заранее
- открыть продукт в отдельной вкладке заранее
- подготовить пользователя с заполненным профилем и рейтингом
