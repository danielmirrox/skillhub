# SkillHub — Implementation Notes

Этот документ фиксирует текущее поведение проекта поверх основной спецификации. Он нужен, чтобы README и релизная документация не расходились с кодом.

## Профили

- GitHub OAuth подставляет `avatar_url` из GitHub, если он доступен.
- Если у пользователя нет фото, UI и backend используют единый дефолтный аватар вместо пустого `img`.
- Импорт из GitHub фильтрует только реальные репозитории, игнорирует profile README и мусорные URL.
- GitHub-статистика участвует в AI-скоринге и подсказках по профилю.

## Поиск и социальные действия

- В поиске доступны избранное и голосование за участников.
- После действия список перезапрашивается или пересортировывается, чтобы счётчики и порядок не расходились с сервером.
- В карточках и публичных профилях есть явная пометка "Это вы", чтобы пользователь не пытался взаимодействовать с собой.

## Команды

- Капитан может удалять участника из своей команды.
- Капитан может закрыть набор.
- Закрытая команда скрывается из общего списка команд и не принимает новые заявки.
- Закрытую команду по-прежнему видит её капитан в своем списке команд.
- Команда может стать недоступной и по другой причине: если свободные слоты закончились, UI показывает "Мест нет".
- При принятии заявки свободный слот уменьшается, при удалении участника слот возвращается.

## Заявки

- Принять заявку можно только если команда активна и у неё есть свободные слоты.
- Если команда закрыта или переполнена, backend возвращает 409, а UI показывает понятное сообщение.
- Статусы заявок в интерфейсе отображаются как человекочитаемые подписи, а не как сырые enum-значения.

## Проверка

- `client` собирается через `npm run build`.
- Типы `client` проходят через `npx tsc --noEmit`.
- Затронутые server-файлы проходят `node --check`.
